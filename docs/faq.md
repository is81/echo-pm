# EchoPM Frequently Asked Questions

## Q: How is EchoPM different from traditional project management tools?

EchoPM is not a replacement for Jira, Trello, or Asana. It is a set of **cognitive tools** — helping you think about the "why" and "what matters most" in project management, rather than managing "who does what by when."

Core differences:
- Traditional tools: Track task status
- EchoPM: Understand project health, priority logic, knowledge structure

## Q: Do I need to know PMBOK to use EchoPM?

No. The PMBOK mapping is provided as a reference for those familiar with it. Each skill is self-contained — you just need to say "help me prioritize" or "generate a project pulse report."

## Q: What do I need to install?

Nothing. All skills' core workflows use only Claude Code's built-in capabilities.

The Python scripts in the `tools/` directory are optional helpers if you want standalone computation power. They also only depend on the Python 3.8+ standard library.

## Q: Can I use just one skill?

Yes. The 7 skills are independent — use them as needed. But they work better together:
- Principles defined by `/project-charter` are referenced by `/retrospective`'s anchor review step
- The knowledge base built by `/knowledge-import` is what `/smart-search` searches
- `/priority-backlog`'s scoring results are referenced by `/project-pulse`'s arousal signals

## Q: How does EchoPM manage its own development?

We use EchoPM to manage EchoPM (dogfooding):
- CHARTER.md was generated by `/project-charter`
- Feature priorities are ranked by `/priority-backlog`
- `/retrospective` is run before every release
- `.echo-pm/pulse.json` tracks project health

## Q: How do I contribute?

1. Fork the repo
2. Choose a skill to improve or create a new one
3. Ensure the new skill follows the patterns in `docs/patterns.md`
4. Submit a PR with a real usage example of that skill
5. Ensure alignment with PMBOK processes (update `docs/pmbok-mapping.md`)

## Q: Why is it called EchoPM?

Echo comes from Project Echo, from which we extracted these patterns. PM stands for Project Management.

"Echo" also hints at the core philosophy: good management is like an echo — you speak your intent, and the system responds with structure. Not control, but resonance.

## Q: Can these patterns really be used for non-software projects?

Yes. Although the patterns originate from software engineering, the core logic is universal:
- Immutable principles: Applicable to any organization with a charter/constitution
- Three-factor priority: Applicable to any scenario requiring multi-dimensional ranking
- Lesson capture: Applicable to any knowledge-intensive work
- Project pulse: Applicable to any team with a concept of "health"
